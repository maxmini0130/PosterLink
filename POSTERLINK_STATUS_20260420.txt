1. 프로젝트 개요
================================================================

PosterLink는 공공 포스터(정부 지원금, 복지, 교육 등) 정보를
모아서 시민에게 맞춤 추천하는 애그리게이션 플랫폼입니다.

- 기술 스택: Next.js (App Router) + Expo (모바일) + Supabase
- 구조: pnpm 모노레포 (apps/web, apps/mobile, packages/*)
- 현재 브랜치: main = develop (동기화 완료, 커밋 5496e2c)
- 총 커밋: 56개
- 웹 빌드: 22개 라우트, 0 에러


2. 완성된 기능 목록
================================================================

[웹 — 사용자]
  - 홈 (맞춤 추천 + 마감 임박 공고)
  - 포스터 목록 (검색, 카테고리/지역 필터, 정렬)
  - 포스터 상세 (이미지, 요약, 링크, D-Day 배지)
  - 찜 (즐겨찾기 추가/삭제/목록)
  - 댓글 (작성/삭제/신고)
  - 알림 센터 (읽음 처리/전체 읽음)
  - 마이페이지 (프로필 수정, 내 댓글)
  - 온보딩 (지역/연령/관심사 3단계)
  - 소셜 로그인 (Google/Kakao OAuth → callback)

[웹 — 운영자(Operator)]
  - 포스터 등록 (이미지 크롭 + OCR 자동 추출)
  - 포스터 목록/수정
  - 검수 요청

[웹 — 관리자(Admin)]
  - 대시보드 (검수 대기/게시 중/신고/사용자 통계)
  - 포스터 검수 (승인/반려)
  - 신고 관리 (댓글 숨김/기각)
  - 기준정보 관리 (카테고리/지역 추가삭제, 유저 권한 변경)
  - 공지 발송 (시스템 알림 전체 발송 + 이력)

[모바일 — 운영자]
  - 로그인 / 카메라 촬영 / 포스터 업로드
  - 푸시 알림 수신 (Expo Push)

[Supabase Edge Functions]
  - process-ocr: 포스터 이미지 → GPT-4o 분석 → 정보 자동 추출
                  + Tavily 웹 검색으로 공식 링크 자동 탐색
  - check-deadlines: 마감 임박 알림 자동 발송 (favorites → notifications → Expo Push)

[DB/백엔드]
  - 통합 스키마 (20260417): 17개 테이블, RLS 정책, 트리거
  - 델타 스키마 (20260418): thumbnail_url, 추천 RPC, 인기 키워드 RPC
  - RPC 함수: get_recommended_posters, search_posters_with_synonyms,
              log_search, get_popular_regions, get_popular_keywords

[테스트]
  - Playwright E2E 테스트 3파일 8케이스 (home, auth, poster)

[공유 패키지]
  - @posterlink/types: 타입 정의 (UserRole, PosterStatus 등)
  - @posterlink/lib: 유틸리티 (getDDay, isDeadlineSoon 등)
  - @posterlink/ui: UI 컴포넌트 (Button)

[커밋 3c15fe6] fix: Gemini 커밋 감사 — 6개 버그 수정
  (1) review_requested → review
      DB CHECK 제약조건에 review_requested가 없어서
      INSERT/UPDATE 시 에러 발생하는 치명적 버그.
      4개 파일 수정 (operator/new, operator/list, admin/posters, admin/dashboard)

  (2) thumbnail_url 컬럼 누락
      코드에서 posters.thumbnail_url을 사용하지만
      어떤 마이그레이션에서도 이 컬럼을 생성하지 않음.
      델타 마이그레이션에 ALTER TABLE 추가.

  (3) reporter_id → reporter_user_id
      CommentSection에서 댓글 신고 시 잘못된 컬럼명 사용.
      DB 컬럼명과 일치하도록 수정.

  (4) poster_favorites → favorites
      check-deadlines Edge Function에서 존재하지 않는
      테이블명 참조. 실제 테이블명 favorites로 수정.

  (5) source_org_name null 크래시
      운영자 포스터 목록에서 .toLowerCase() 호출 시
      null이면 런타임 크래시. null guard 추가.

  (6) PosterStatus 타입 불일치
      packages/types의 타입 정의가 DB 제약조건과 불일치.
      closed, archived 추가, 없는 값 제거.

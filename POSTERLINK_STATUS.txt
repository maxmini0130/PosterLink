[PosterLink 프로젝트 개발 현황 리포트 - 2026.04.16]

==================================================
1. 오늘 완료된 작업 (Phase 1 MVP 95% 달성)
==================================================

[인프라 및 환경]
- pnpm workspaces 기반 모노레포 구조 구축 (Web, Mobile, Shared Packages)
- Supabase DB 스키마 설계 및 마이그레이션 완료 (profiles, posters, comments 등)
- 서비스 실행 포트 변경 (3000 -> 4000) 및 Tailwind CSS 설정 최적화

[사용자(User) 기능]
- 실데이터 연동 온보딩 플로우 (지역, 연령, 관심사 설정)
- 맞춤형 홈 피드 (사용자 조건 일치 공고 및 마감 임박 섹션)
- 통합 검색 및 필터링 시스템 (최근 검색어, 인기 키워드 기능 포함)
- 포스터 상세 정보 및 실시간 찜하기(Favorite) 기능
- 질문/후기 댓글 작성 및 삭제, 부적절 댓글 신고 기능
- 알림 센터 UI 및 기기 푸시 토큰 수집 로직

[운영자(Operator) 기능]
- 포스터 등록 폼 고도화 (Supabase Storage 연동 이미지 업로드)
- 이미지 보정 도구 통합 (업로드 즉시 자르기, 회전, 확대/축소 가능)
- 운영자 전용 포스터 관리 목록 및 검수 요청 시스템
- Expo 기반 모바일 현장 촬영 및 즉시 업로드 앱 초기 버전

[관리자(Admin) 기능]
- 관리자 전용 대시보드 및 보안 레이아웃
- 포스터 검수 및 원클릭 승인/반려 시스템
- 댓글 신고 센터 (신고 검토 후 댓글 숨김 처리 기능)
- 마스터 데이터 관리 (카테고리 및 지역 코드 실시간 추가/삭제)

==================================================
2. 향후 작업이 필요한 내용 (Phase 2)
==================================================

- 소셜 로그인 연동: 카카오/구글 로그인 Supabase 설정 및 연결
- 알림 자동화 서버: 마감 1일 전 자동 알림을 쏴주는 Supabase Edge Function 배포
- 실제 서비스 배포: Vercel(웹) 및 EAS(모바일)를 통한 운영 환경 구축
- OCR 자동화: 포스터 업로드 시 텍스트 자동 추출 및 요약 기능 (이월 과제)
- 다크 모드 지원: 시스템 설정에 따른 UI 테마 대응

==================================================
3. 운영 및 테스트 시 중요 체크사항 (필독)
==================================================

[Supabase 설정 필수 확인]
- Auth 설정: 편리한 가입 테스트를 위해 [Authentication > Providers > Email]에서 'Confirm email' 옵션을 OFF로 설정 권장.
- Storage 권한: 'poster-originals' 버킷이 'Public'으로 설정되어야 이미지 조회가 가능함.
- RLS 정책: 현재 설정된 Row Level Security 정책이 실제 운영 시나리오와 일치하는지 재점검 필요.

[로컬 환경 변수]
- apps/web/.env.local 파일 내 NEXT_PUBLIC_SUPABASE_URL 및 ANON_KEY가 본인의 프로젝트 값과 일치해야 함.
- 포트 번호는 http://localhost:4000 을 기준으로 설정되어 있음.

[모바일 앱 실행]
- Expo Go 앱을 통해 실기기 테스트 시, PC와 스마트폰이 같은 Wi-Fi 환경에 있어야 함.
- 실제 푸시 알림 수신은 시뮬레이터가 아닌 '실기기'에서만 정상 작동함.

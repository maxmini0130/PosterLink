# PosterLink 작업 정리 - 2026-07-23

## 완료된 작업

- 수집 기관 운영 체계를 만들었습니다.
  - 수집 기관 목록, 상태, 우선순위, 최근 성공 시각, 최근 수집 결과를 관리자 화면에서 볼 수 있습니다.
  - GitHub Actions를 통해 단일 기관 또는 선택한 여러 기관을 백그라운드로 실행할 수 있습니다.
  - 실행 요청은 `collection_source_runs`에 `running` 상태로 기록됩니다.

- 수집 실행 이력과 진단 화면을 강화했습니다.
  - 최근 수집 이력에서 확인/신규/유효/중복/제외/실패 수를 볼 수 있습니다.
  - 실행 상세에서 제외 사유, 제외 샘플, 첨부파일 분석 샘플, 원문 추적 샘플을 볼 수 있습니다.
  - 기관별 진단 영역에서 설정 요약, 규칙 점검, 권장 조치를 확인할 수 있습니다.

- 수집 알림과 지연 실행 대응을 추가했습니다.
  - `/admin/notifications`에서 수집 오류와 오래 끝나지 않은 실행을 알림으로 볼 수 있습니다.
  - `/admin/collection-sources?source=기관코드`로 들어오면 해당 기관을 자동 검색, 선택, 스크롤, 펼침 처리합니다.
  - `/admin/collection-sources` 상단에 `실행 지연` 지표를 추가했습니다.
  - 오래 `running`으로 남은 실행은 `완료 지연`으로 표시됩니다.
  - 관리자 화면에서 `지연 종료 처리`로 오래 남은 실행 이력을 오류 상태로 닫을 수 있습니다.

- 크롤러 품질 필터를 강화했습니다.
  - 주민등록, 민방위, 대피소, 생활폐기물 같은 행정 안내성 글을 더 잘 제외하도록 했습니다.
  - 결과 발표, 선정자 명단, 오래된 공고, 단순 일정표 등 포스터링크 대상이 아닌 글을 더 많이 거르도록 했습니다.
  - 웹접근성 마크, 로고, 배너, 비포스터 이미지를 가져오지 않도록 검증을 강화했습니다.

- 기관/어댑터를 추가하고 튜닝했습니다.
  - 마포 일자리/노동 관련 어댑터를 추가했습니다.
  - 마포문화재단, 마포구, 마포구고용복지지원센터, 청년몽땅정보통, 온통청년, K-Startup 등 운영 수집 흐름을 점검했습니다.
  - 중앙/지역 수집원 seed migration을 추가했고, 사용자가 관련 SQL을 실행했습니다.

- 이미지 없는 공고 후보 관리 흐름을 추가했습니다.
  - 포스터 이미지는 없지만 공고 가치가 있는 글은 별도 후보로 분리됩니다.
  - `/admin/notice-candidates`에서 후보를 검색, 편집, 이미지 업로드, 제작중, 완료, 제외, 삭제 처리할 수 있습니다.
  - 향후 AI 포스터 생성 기능을 붙일 수 있는 운영 기반이 생겼습니다.

- 경유 페이지에서 실제 원문을 추적하는 기반을 추가했습니다.
  - 청년몽땅정보통처럼 포털 페이지가 실제 기관 원문으로 이어지는 경우를 처리할 수 있게 했습니다.
  - 수집 메타데이터에 원문 추적 성공/실패 샘플을 남겨 검수할 수 있습니다.

- 첨부파일/OCR 기반 검증의 기초를 추가했습니다.
  - 첨부파일 본문 분석 흐름을 추가했습니다.
  - 포스터 OCR 인사이트를 검증 흐름에 반영했습니다.
  - 검수 화면에서 수집 근거를 더 잘 확인할 수 있게 했습니다.

- 방문 통계 기반을 추가했습니다.
  - `site_visit_logs` migration을 추가했고 사용자가 실행했습니다.
  - 관리자 통계 화면을 추가했습니다.
  - 유입 경로, 랜딩 페이지, 기기/앱 비중을 확인할 수 있는 기반을 만들었습니다.
  - 같은 사용자 또는 같은 IP를 중복 방문자로 과하게 세지 않도록 집계 방향을 조정했습니다.

- 검색엔진 소유권 확인 파일을 반영했습니다.
  - Google 소유권 확인 HTML 파일을 루트에 추가했습니다.
  - Naver 소유권 확인 HTML 파일을 루트에 추가했습니다.
  - 사이트맵 제출 위치와 확인 방법을 안내했습니다.

- 메인 문구와 모바일 가독성을 조정했습니다.
  - 데스크톱 상단 문구는 한 줄로 보이도록 조정했습니다.
  - 모바일에서는 자연스럽게 여러 줄로 나뉘도록 조정했습니다.

  - 공공기관, 지자체, 청년센터, 창업기관, 복지기관, 문화재단, 교육기관, 도서관, 채용기관을 우선순위대로 추가합니다.
  - 기관 수를 무작정 늘리기보다 유효 공고 비율을 확인하면서 확장합니다.

  - 목록 셀렉터, 상세 셀렉터, 이미지 셀렉터, 첨부파일 셀렉터, 페이지네이션, URL 필터, 원문 추적 규칙을 기관별로 조정합니다.
  - 실제 운영 수집 결과를 보고 제외 사유와 샘플을 기준으로 개선합니다.

- 수집 기관 설정 UI를 더 쉽게 만들어야 합니다.
  - 현재 JSON 설정 중심인 부분을 구조화된 입력 UI로 바꿉니다.
  - adapter, 게시판 목록, 셀렉터, URL 필터, 첨부파일, 원문 추적 옵션을 안전하게 편집할 수 있게 합니다.
  - 저장 전 설정 검증 기능을 추가합니다.

- 첨부파일/PDF/HWP 분석을 강화해야 합니다.
  - PDF/HWP/HWPX에서 모집기간, 대상, 혜택, 신청방법, 신청 URL을 더 안정적으로 추출합니다.
  - 실패 사유를 더 명확하게 관리자 화면에 보여줍니다.

- OCR 기반 포스터 내용 추출을 고도화해야 합니다.
  - 포스터 이미지 안의 모집기간, 대상, 장소, 신청방법, 문의처를 구조화합니다.
  - 웹접근성 마크, 로고, 배너, 비포스터 이미지는 계속 차단합니다.

- 이미지 없는 공고용 AI 포스터 생성 기능을 만들어야 합니다.
  - API 비용 관리를 위해 관리자 수동 실행 방식으로 시작합니다.
  - 후보 내용을 검토한 뒤 AI 포스터 초안을 생성하고, 운영자가 최종 확인 후 게시합니다.
  - 생성 프롬프트, 생성 상태, 비용 메타데이터, 최종 이미지 URL을 기록합니다.

- 이미지 없는 공고 후보 운영을 더 편하게 만들어야 합니다.
  - 후보 일괄 처리, 중복 비교, 기관별 묶음, 상태별 빠른 필터를 강화합니다.
  - 후보에서 포스터 검수 항목으로 전환하는 흐름을 더 명확하게 만듭니다.

- 중복 공고 판단과 병합을 강화해야 합니다.
  - 원문 URL, 기관명/제목 유사도, 모집기간, 첨부파일 해시, 포스터 이미지 해시, 본문 임베딩, 신청 URL을 함께 비교합니다.
  - 같은 공고는 새 글로 만들지 말고 대표 공고에 출처를 추가하는 구조가 필요합니다.

- 사용자 제보 기능의 전체 운영 플로우가 필요합니다.
  - 공고 URL 제보, 포스터 이미지 업로드, 누락 기관 제보, 마감/오류 신고, 중복 공고 신고를 받습니다.
  - 바로 게시하지 않고 운영자 검수로 보내는 방식이 안전합니다.

- 기관 담당자용 페이지는 아직 남아 있습니다.
  - 기관 인증, 공고 직접 등록, 기존 공고 수정, 마감 처리, 성과 확인 기능이 필요합니다.
  - 조회, 저장, 원문 클릭, 신청 클릭 통계를 기관에 제공할 수 있게 합니다.

- 자동 진단/알림을 더 발전시켜야 합니다.
  - 반복 실패, 구조 변경, 장기 미수집, 유효율 급락, 필수정보 누락률 증가를 자동 감지합니다.
  - 알림에서 바로 해당 기관과 실행 이력으로 이동하도록 유지합니다.

- 운영 테스트를 한 번에 진행해야 합니다.
  - Search Console/Naver 소유권 확인, 사이트맵 접근, 방문 통계, 유입 경로, 모바일 표시를 확인합니다.
  - 크롤러 실행, 수집 이력, 지연 종료 처리, 검수 화면, 후보 관리, 포스터 승인, 메인 노출까지 전체 시나리오로 테스트합니다.

- 모바일 앱 작업은 아직 남아 있습니다.
  - iOS 앱은 아직 만들지 않았습니다.
  - 당장은 PWA/모바일 웹 테스트를 먼저 안정화하는 것이 좋습니다.

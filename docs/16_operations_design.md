# PosterLink 운영설계.md

## 문서 목적
이 문서는 포스터링크(PosterLink)를 단순한 공공 포스터 모음 서비스가 아니라,
유연한 정보구조, 정확한 탐색, 신뢰 기반의 장기 사업 확장성을 가진 실제 서비스로 운영하기 위한 기준안을 정리한다.

핵심 범위는 아래 3가지다.

1. 정보구조 운영 설계
2. 검색 및 개인화 설계
3. 수익화 가설 설계

---

## 1. 정보구조 운영 설계

### 1.1 기본 원칙

#### 원칙 1. 분류체계는 하드코딩하지 않는다
카테고리, 대상 조건, 태그, 추천 가중치, 동의어 사전은 코드에 고정하지 않고 운영자가 관리자 화면에서 수정 가능해야 한다.

#### 원칙 2. 포스터 이미지를 보여주는 것이 아니라 공고 정보를 구조화한다
서비스의 본질은 이미지 뷰어가 아니라 공공정보 탐색 플랫폼이다. 따라서 제목, 기관, 지역, 대상, 마감일, 링크, 상태값이 구조화되어야 한다.

#### 원칙 3. 하나의 공고에 다중 조건이 연결될 수 있어야 한다
예를 들어 하나의 공고는 다음을 동시에 가질 수 있다.
- 지역: 서울특별시
- 대상: 청년
- 조건: 1인가구
- 카테고리: 주거/생활
- 태그: 온라인접수, 신청간편

#### 원칙 4. 공식 정보와 운영 정보를 분리한다
- 공식 정보: 기관명, 공식 링크, 신청 링크, 원문 내용, 마감일, 문의처
- 운영 정보: 운영 요약, 태그, 추천 분류, 동의어, 검색 확장어, 우선 노출 설정

---

### 1.2 정보구조 4계층

#### A. 기준 마스터 정보
운영자가 관리하는 기준 데이터.
- 카테고리
- 지역
- 대상 그룹
- 기관 유형
- 공고 상태
- 링크 유형
- 태그 사전
- 동의어 사전

#### B. 공고 본문 정보
개별 포스터/공고에 귀속되는 핵심 데이터.
- 제목
- 부제
- 기관명
- 포스터 이미지
- 공식 링크
- 신청 링크
- 마감일
- 본문 요약
- 상세 설명
- 문의처
- 등록 상태

#### C. 조건/분류 정보
검색과 추천을 위해 공고에 연결되는 구조화 정보.
- 지역 조건
- 연령 조건
- 성별 조건
- 대상 조건
- 카테고리
- 운영 태그
- 검색 확장어
- 공식성 등급

#### D. 사용자/개인화 정보
사용자별 탐색 및 추천에 필요한 데이터.
- 기본 지역
- 연령대
- 관심 카테고리
- 관심 태그
- 찜 내역
- 클릭 내역
- 검색 이력
- 알림 설정

---

### 1.3 카테고리 운영안

초기에는 1차 카테고리와 2차 카테고리 구조로 설계한다.

#### 1차 카테고리
- 지원금/복지
- 교육/강좌
- 행사/축제
- 문화/체험
- 일자리/창업
- 주거/생활
- 건강/의료
- 가족/육아
- 참여/모집
- 공공시설/프로그램

#### 2차 카테고리 예시

지원금/복지 아래
- 청년지원
- 생활지원
- 긴급지원
- 응시료 지원
- 바우처
- 돌봄지원

교육/강좌 아래
- 취업교육
- 디지털교육
- 취미강좌
- 건강교육
- 부모교육

주거/생활 아래
- 월세지원
- 생활비지원
- 공공임대
- 주거상담
- 생활안전

#### 운영 규칙
- 한 공고는 1차 카테고리 1개를 기본으로 가진다.
- 필요 시 2차 카테고리 다중 연결을 허용한다.
- 부족한 분류는 운영 태그로 보완한다.
- 카테고리는 추가/수정/비활성화가 가능해야 한다.

---

### 1.4 지역 구조 운영안

지역은 반드시 트리 구조로 설계한다.

#### 권장 구조
- 전국
- 시/도
- 시/군/구
- 읍/면/동
- 생활권/기관권역 태그(선택)

#### 예시
- 전국
- 서울특별시
- 서울특별시 강남구
- 서울특별시 강남구 역삼동

#### 운영 규칙
- 전국 대상 공고는 별도 플래그를 둔다.
- 상위 지역 공고와 하위 지역 공고를 구분한다.
- 사용자가 상위 지역만 선택해도 하위 지역 공고 일부를 추천할 수 있게 한다.
- 기관 단위 행사는 필요 시 생활권 태그를 추가할 수 있다.

---

### 1.5 대상 조건 운영안

대상 조건은 단순 연령대만으로 처리하지 않는다.

#### 기본 대상 그룹
- 전체
- 아동
- 청소년
- 청년
- 중장년
- 시니어
- 여성
- 남성
- 가족
- 1인가구
- 학생
- 구직자
- 직장인
- 소상공인
- 자영업자
- 임산부/부모
- 장애인
- 다문화가정
- 외국인

#### 세부 조건 속성
- 연령 최소값 / 최대값
- 성별 제한 여부
- 거주 제한 여부
- 소득 제한 여부
- 직업/신분 조건
- 가구 형태
- 추가 자격 조건

#### 운영 규칙
- 표시값과 실제 조건 수치를 같이 저장한다.
- 예: 표시값 = 청년 / 내부값 = 19세 이상 39세 이하
- 기관마다 청년 기준이 다를 수 있으므로 라벨과 숫자 범위를 분리한다.

---

### 1.6 태그와 동의어 운영안

#### 운영 태그
운영자가 붙이는 탐색 보조 태그.
- 신청간편
- 온라인접수
- 선착순
- 무료
- 가족참여
- 주말행사
- 반복모집
- 현장접수
- 추천공고
- 마감임박

#### 동의어/유사어 사전
검색 확장용 사전.
- 어르신 = 시니어 = 노인
- 지원금 = 보조금 = 지원
- 월세 = 주거비 = 임차료
- 취업 = 일자리 = 구직
- 행사 = 축제 = 프로그램
- 교육 = 강좌 = 수업

#### 운영 규칙
- 태그는 운영자가 직접 관리한다.
- 동의어 사전은 운영자 관리 + 검색 로그 분석으로 확장한다.
- 반복되는 사용자 검색 표현은 사전에 추가한다.

---

### 1.7 공고 상태값 운영안

#### 권장 상태값
- `draft`: 초안
- `review`: 검수중
- `published`: 게시중
- `hidden`: 숨김
- `closed`: 마감
- `archived`: 보관

#### 운영 규칙
- 마감일 경과 시 자동으로 `closed` 처리한다.
- 운영 이슈 발생 시 `hidden` 처리한다.
- 장기 보관용은 `archived`로 전환한다.
- 상태 변경 이력은 로그로 남긴다.

---

### 1.8 관리자에서 유연하게 관리해야 할 항목

관리자 화면에서 아래는 반드시 수정 가능해야 한다.
- 카테고리 추가/수정/비활성화
- 지역 트리 관리
- 대상 그룹 관리
- 태그 관리
- 동의어/검색 확장어 관리
- 추천 가중치 관리
- 공고 상태 관리
- 공식 링크/참고 링크 유형 관리
- 노출 우선순위 조정
- 수정 이력 조회

## 2. 검색 및 개인화 설계안

### 2.1 검색 목표

포스터링크 검색의 목표는 단순 문자열 매칭이 아니다.

#### 목표
- 사용자가 생활언어로 검색해도 원하는 공고를 찾게 한다.
- 비슷한 표현으로 검색해도 의미적으로 맞는 결과가 나온다.
- 공공정보 특성상 정확성과 신뢰성을 유지한다.
- 내 조건에 맞는 공고가 우선 정렬된다.

핵심 문장:
**AI처럼 똑똑하게 느껴지되, 행정정보처럼 정확해야 한다.**

---

### 2.2 검색 방식: 하이브리드 검색

실서비스는 3단계 검색 구조를 권장한다.

#### 1단계. 구조 필터
정확하게 걸러야 하는 조건은 먼저 필터링한다.
- 지역
- 마감 여부
- 카테고리
- 신청 가능 조건
- 공식성 여부

#### 2단계. 의미 기반 탐색
사용자 검색어를 의미적으로 확장한다.

예시:
- 어르신 운동 → 시니어, 건강, 체조, 운동교실
- 청년 월세 → 청년, 주거, 월세, 임차료, 주거지원
- 아이랑 행사 → 가족, 아동, 체험, 행사, 주말

#### 3단계. 재정렬
후보 결과를 사용자 적합도 기준으로 재정렬한다.
- 의미 유사도
- 지역 적합도
- 관심 카테고리 일치
- 대상 조건 일치
- 마감 임박도
- 공식성/신뢰도
- 최신성
- 찜/클릭 이력

---

### 2.3 검색 랭킹 개념

초기 랭킹은 규칙 기반 점수화로 시작한다.

#### 예시 가중치
- 의미 유사도: 35
- 지역 일치: 20
- 관심 카테고리 일치: 15
- 대상 조건 일치: 15
- 마감 임박: 5
- 최신 등록: 5
- 공식성/운영 검수 완료: 5

#### 운영 원칙
- 초기에는 룰 기반으로 시작한다.
- 이후 로그가 쌓이면 CTR, 찜률, 공식 링크 클릭률을 반영한다.
- 추천이 과도하게 개인화되지 않도록 전체 중요 공고를 일부 섞는다.

---

### 2.4 검색 UX 운영안

검색 품질은 엔진과 UX가 함께 만들어진다.

#### 꼭 필요한 UX 요소
- 추천 검색어
- 인기 검색어
- 유사 검색어 제안
- 빠른 필터 영역
- 검색 결과 없음 시 대체 제안
- 비슷한 공고 섹션
- 전국 대상 공고 함께 보기

#### 검색 결과 화면 구조
- 상단: 검색어 + 필터 요약
- 중단: 가장 적합한 결과
- 하단: 비슷한 공고 / 마감 임박 / 지역 유사 공고

---

### 2.5 개인화 추천 운영안

개인화는 가입정보만이 아니라 행동 데이터까지 같이 본다.

#### 개인화 입력값
- 지역
- 연령대
- 관심 카테고리
- 관심 태그
- 찜
- 클릭
- 검색어
- 알림 설정

#### 추천 피드 구성
- 내 조건에 맞는 공고
- 내 지역 신규 공고
- 찜한 공고와 유사한 공고
- 마감 임박 공고
- 전체 대상이지만 반응이 좋은 공고

#### 운영 원칙
- 맞춤 추천만 보여주면 정보 편식이 생길 수 있다.
- 권장 비율은 맞춤 70 / 전체 중요 공고 30 수준이다.

---

### 2.6 검색/추천 품질 평가 지표

#### 검색 핵심 지표
- 검색 결과 클릭률
- 검색 후 이탈률
- 검색 후 찜 전환율
- 검색 후 공식 링크 클릭률
- 검색어 재입력 비율
- 결과 없음 비율
- 유사어 제안 클릭률

#### 추천 핵심 지표
- 홈 피드 클릭률
- 찜 전환율
- 공식 링크 이동률
- 재방문율
- 사용자별 추천 적중도

---

## 3. 수익화 가설 설계안

### 3.1 수익화 기본 원칙

포스터링크는 너무 빠른 수익화보다 사용자 가치와 데이터 축적을 우선한다. 다만 수익화 구조는 지금부터 설계해 둔다.

#### 원칙 1. 핵심 공공정보는 무료 접근을 유지한다
공공정보를 찾고 확인하는 핵심 경험은 유료 장벽으로 막지 않는다.

#### 원칙 2. 수익모델이 정보 신뢰를 훼손하면 안 된다
유료 노출이 공식 정보보다 우선하거나 검색 결과를 왜곡해서는 안 된다.

#### 원칙 3. 수익모델은 사용자 가치 위에 얹는다
먼저 “다시 찾기 쉽다”는 경험이 만들어져야 한다.

#### 원칙 4. 상품화는 늦춰도 로그 설계는 지금부터 한다
나중에 판매할 상품은 지금부터 데이터가 쌓여야 한다.

---

### 3.2 수익화 단계별 가설

#### 1단계. 사용자 가치 검증 단계
강한 수익화 없음.

목표:
- 사용자 탐색 습관 형성
- 검색/찜/알림 정착
- 운영 품질 확보

필요 데이터:
- 조회수
- 찜수
- 공식 링크 클릭수
- 검색 로그
- 지역/카테고리별 반응

#### 2단계. 기관 가치 검증 단계
시범적으로 기관 관점 기능 제공.

가능 모델:
- 기관용 성과 리포트
- 기관 소개형 페이지
- 기관 전용 등록 계정 시험 운영
- 특정 지역/기관 대상 PoC

목표:
- 기관이 어떤 데이터를 원하나
- 어떤 지표를 도입 효과로 보나
- 어떤 업무가 줄어드나 확인

#### 3단계. 상품화 단계
실제 판매 가능한 모델 오픈.

가능 상품:
- 기관용 등록/관리 SaaS

### 3.4 수익모델 후보군

#### A. 기관용 SaaS
가장 유력한 장기 모델.

제공 기능:
- 포스터 등록/관리
- 공고 현황 관리
- 성과 대시보드
- 링크 클릭 리포트
- 카테고리/지역 성과 분석

#### B. 캠페인 패키지
특정 기간 집중 홍보 상품.

예시:
- 청년정책 집중홍보
- 여름 행사 큐레이션
- 복지 안내 주간

#### C. 기관 전용 페이지/프로필
기관별 공고 모아보기, 담당자 관리 기능.

#### D. 데이터 리포트
- 월간 지역 관심사 리포트
- 카테고리별 반응 리포트
- 공고 반응 비교 리포트

#### E. API/연동
향후 공공기관 홈페이지, 챗봇, 앱에 연결 가능한 정보 API.

## 6. 요약 결론

포스터링크는 아래 3개가 핵심이다.

1. **운영 가능한 정보 구조**
2. **정확 필터 + 의미 검색 + 개인화 재정렬의 하이브리드 탐색**
3. **신뢰를 해치지 않는 단계적 수익화 구조**

즉, 포스터링크는 단순한 공고 앱이 아니라,
**유연하게 운영되는 공공정보 탐색 플랫폼**으로 설계되어야 한다.

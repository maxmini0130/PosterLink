# 01. 사용자 페르소나 및 가치 정의

## 페르소나 1: 지역 혜택 탐색형 사용자

- 연령: 20대~40대
- 특징: 지역 지원금, 교육, 행사 정보를 놓치고 싶지 않음
- 문제:
  - 포스터는 봤지만 저장하지 못함
  - 홈페이지를 다시 찾기 번거로움
  - 여러 기관 사이트를 돌아다니기 싫음
- 기대 가치:
  - 지역 중심 추천
  - 마감 임박 공고 우선 확인
  - 찜/알림으로 재탐색 비용 절감

## 페르소나 2: 특정 조건 대상자

예시:
- 청년
- 시니어
- 소상공인
- 학부모
- 1인가구
- 구직자
- 장애인 관련 정책 대상자

### 니즈
- “나에게 해당되는 것만 보고 싶다”
- “전국민 대상과 내 조건 대상 공고를 구분해 보고 싶다”
- “복잡한 자격 조건을 쉽게 이해하고 싶다”

## 페르소나 3: 운영자

- 역할: 포스터 수집 및 등록
- 문제:
  - 사진 촬영 품질 편차
  - 링크 탐색 시간이 많이 듦
  - 포스터 정보 구조화가 번거로움
- 기대 가치:
  - 촬영 후 빠른 등록
  - 이미지 보정
  - OCR 자동 추출
  - 카테고리/지역/대상 조건 입력 편의성
  - 검수/승인 프로세스 명확화

## 페르소나 4: 관리자

- 역할: 품질/정책 운영
- 문제:
  - 허위성 댓글, 광고성 댓글, 분쟁성 댓글 관리 필요
  - 포스터 만료/오류 링크/중복 포스터 정리 필요
- 기대 가치:
  - 신고/숨김/삭제 일괄 처리
  - 포스터 상태 관리
  - 금칙어/제재 정책
  - 운영 로그

## 대표 사용 시나리오

### 시나리오 A: 사용자가 포스터를 보고 나중에 다시 찾음
1. 사용자가 길에서 구청 포스터를 봄
2. 나중에 앱을 열어 지역/카테고리 기준으로 탐색
3. 관련 포스터를 찾음
4. 상세에서 공식 링크를 확인
5. 찜 후 마감 전 다시 확인

### 시나리오 B: 가입 즉시 맞춤 추천을 받음
1. 사용자가 가입
2. 지역, 연령대, 관심 카테고리 선택
3. 홈에서 본인 조건에 맞는 포스터 우선 노출
4. 찜, 댓글, 상세 조회 진행

### 시나리오 C: 운영자가 포스터를 등록
1. 운영자가 현장에서 포스터 촬영
2. 이미지 보정본 확인
3. OCR 결과 검수
4. 공식 링크 연결
5. 카테고리/지역/조건 입력
6. 게시 승인

### 시나리오 D: 사용자 댓글 신고 처리
1. 사용자가 댓글 작성
2. 다른 사용자가 허위정보 또는 광고성 댓글 신고
3. 관리자가 신고 내용을 검토
4. 댓글 숨김 또는 삭제 처리
5. 필요 시 작성자 경고/제재

## 가치 제안 문장

### 사용자 관점
- “길에서 본 공공 포스터를 다시 찾을 수 있다.”
- “내 조건에 맞는 공고가 먼저 보인다.”
- “긴 공고문 대신 핵심만 빠르게 이해할 수 있다.”

### 운영자 관점
- “포스터를 빠르게 등록하고 구조화할 수 있다.”
- “공식 링크와 조건 정보를 체계적으로 연결할 수 있다.”

### 관리자 관점
- “서비스 신뢰를 해치지 않도록 댓글/포스터를 운영정책 기준으로 관리할 수 있다.”

# 08. MVP 범위 및 로드맵

## 1. MVP 목표

초기 MVP의 목표는 “전국 모든 포스터를 자동 수집”이 아니다.  
핵심은 아래 3가지다.

1. 운영자가 직접 포스터를 등록할 수 있다.
2. 사용자가 내 조건에 맞는 포스터를 볼 수 있다.
3. 사용자가 찜하고 다시 찾아올 수 있다.

## 2. MVP 포함 범위

### 운영자 측
- 포스터 이미지 업로드
- 보정본 확인
- OCR 텍스트 검수
- 포스터 기본 메타데이터 입력
- 공식 링크 입력
- 게시/비공개/마감 상태 관리

### 사용자 측
- 로그인/회원가입
- 지역/연령대/관심 카테고리 설정
- 추천 피드
- 탐색/검색
- 포스터 상세
- 찜하기
- 기본 댓글
- 신고하기
- 알림(기본형)

### 관리자 측
- 포스터 목록/상태 관리
- 댓글/신고 관리
- 금칙어 관리(기본형)

## 3. MVP 제외 범위

- 기관 직접 등록 포털
- 사용자 포스터 제보
- 복잡한 AI 추천
- OCR 자동 완성 고도화
- 댓글 좋아요/랭킹
- 포인트/배지 시스템
- 실시간 채팅
- 광고 시스템
- 결제 기능
- 통계 대시보드 고도화

## 4. 단계별 로드맵

### Phase 1: 코어 구축
- 인증
- 사용자 프로필
- 기준정보
- 포스터 CRUD
- 사용자 홈/탐색/상세
- 찜
- 댓글
- 관리자 기초 CMS

### Phase 2: 운영 효율화
- OCR 연동
- 이미지 보정 워크플로우
- 링크 관리 개선
- 만료 자동 처리
- 알림 개선
- 댓글 유형 분리

### Phase 3: 개인화 고도화
- 추천 점수 정교화
- 유사 포스터 추천
- 찜 기반 추천
- 관심 조건 세분화
- 인기순/반응지표 반영

### Phase 4: 확장
- 기관 직접 등록
- 사용자 제보
- 기관용 성과 대시보드
- 포스터 성과 분석
- 외부 채널 배포 연동

## 5. 우선순위

### 반드시 먼저 개발
1. 기준정보/DB
2. 포스터 등록/조회
3. 사용자 개인화 조건 저장
4. 추천 홈
5. 포스터 상세
6. 찜
7. 댓글 + 신고
8. 관리자 처리

### 나중에 개발
1. OCR 자동화 고도화
2. 이미지 보정 자동화
3. 유사 공고 추천
4. 기관 포털
5. 성과 분석

## 6. MVP 완료 기준

### 기능 기준
- 운영자가 포스터 1건을 처음부터 끝까지 등록 가능
- 사용자가 관심 조건 설정 후 홈에서 포스터 확인 가능
- 사용자가 포스터를 찜하고 해제 가능
- 댓글 작성/삭제/신고 가능
- 관리자가 신고 처리 가능

### 품질 기준
- 모바일 홈/상세 기본 사용성 확보
- 등록된 포스터에 공식 링크 최소 1개 존재
- 마감된 포스터는 기본 추천 피드에서 제외

## 1. 카카오 로그인 PKCE 오류 수정

**문제:** 카카오 OAuth 콜백에서 "PKCE code verifier not found in storage" 오류 발생.

**원인:** `app/auth/callback/page.tsx`가 클라이언트 컴포넌트로 구현되어 있어 PKCE verifier를 localStorage에 저장했으나, 카카오 OAuth 리디렉션 후 cross-origin 이동으로 localStorage가 초기화되었다.

**해결:**
- `page.tsx` 삭제, `route.ts` (서버사이드 Route Handler)로 교체
- `createServerClient` + 쿠키 기반 세션 저장으로 PKCE verifier 유지
- `pendingMap` 패턴으로 `exchangeCodeForSession` 직후 세팅된 쿠키를 이후 쿼리에서 즉시 참조 가능하게 처리

관련 파일: `apps/web/app/auth/callback/route.ts`

## 2. 온보딩 저장 서버사이드 API 분리

**문제:** 온보딩 완료 시 `user_interest_categories` FK 제약 오류 반복 발생.

**원인:** 클라이언트에서 직접 Supabase를 호출할 때 RLS 정책 충돌 및 세션 불일치.

**해결:**
- `/api/onboarding/complete` Route Handler 신규 생성
- `SUPABASE_SERVICE_ROLE_KEY`로 admin 클라이언트를 사용해 RLS 우회
- `profiles` upsert → `user_interest_categories` delete/insert 순서 보장

관련 파일: `apps/web/app/api/onboarding/complete/route.ts`, `apps/web/app/onboarding/page.tsx`

## 3. Sentry DSN 플레이스홀더 오류 수정

**문제:** Vercel 환경변수에 실제 DSN 대신 문자열 `"NEXT_PUBLIC_SENTRY_DSN"`이 등록되어 콘솔 오류 발생.

**해결:** `rawDsn?.startsWith("https://")` 검증 추가. 유효한 URL이 아니면 Sentry 초기화 건너뜀.

관련 파일: `apps/web/instrumentation.ts`, `apps/web/instrumentation-client.ts`

## 4. get_recommended_posters 함수 타입 버그 수정

**문제:** `POST /rpc/get_recommended_posters 400 Bad Request`.

**원인 1 (이전 세션):** 마이그레이션이 운영 DB에 적용되지 않아 함수 자체가 없었음 → SQL Editor에서 직접 적용.

**원인 2 (이번 세션):** `posters` 테이블의 `title`, `source_org_name`, `summary_short`, `thumbnail_url` 컬럼이 `varchar(255)` 타입인데, 함수 반환 타입이 `text`로 선언되어 PostgreSQL 타입 불일치 오류 발생.

**해결:** SELECT 절에 `::TEXT` 캐스팅 명시. `supabase db query --linked -f` 로 적용.

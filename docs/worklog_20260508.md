## 1. 포스터 등록 요청 + 포인트 시스템

### 배경
관리자만 포스터를 등록할 수 있던 구조에서, 일반 사용자도 발견한 포스터를 요청하면 관리자가 검수 후 등록하는 구조로 확장.
승인 시 요청자에게 50포인트 지급.

### DB 마이그레이션
**파일:** `supabase/migrations/20260508000000_add_poster_requests_and_points.sql`

- `poster_requests` 테이블 생성
  - `user_id`, `description`, `location_text`, `image_url`, `status` (pending/approved/rejected), `admin_note`
- `point_logs` 테이블 생성
  - `user_id`, `amount`, `reason`, `ref_id`
- `profiles.points INTEGER DEFAULT 0` 컬럼 추가
- Storage 버킷 `poster-requests` 생성 (RLS: 본인만 업로드)

### 신규 페이지
- **`apps/web/app/posters/request/page.tsx`** — 사용자 등록 요청 폼
  - 사진 업로드 (선택), 장소 텍스트 (선택), 설명 (필수)
  - 50P 인센티브 배너 표시
- **`apps/web/app/admin/requests/page.tsx`** — 관리자 검수 페이지
  - 탭: 대기중 / 승인됨 / 반려됨
  - 승인 시: `poster_requests.status` 업데이트 + `point_logs` 삽입 + `profiles.points += 50`
  - 반려 시: 사유 입력 모달

### 기존 파일 수정
- **`apps/web/app/admin/layout.tsx`** — "등록 요청" 메뉴 추가 (Inbox 아이콘)
- **`apps/web/app/mypage/page.tsx`** — 포인트 배지 표시, "포스터 등록 요청하기" 메뉴
- **`apps/web/app/page.tsx`** — 로그인 사용자에게 등록 요청 배너 카드 표시

## 3. 맞춤 설정 전면 개선 (개인화)

### 배경
신규 사용자가 온보딩 없이 바로 앱에 접근 가능했고, 추천 알고리즘에 성별이 반영되지 않았던 문제 해결.

### DB 마이그레이션
**파일:** `supabase/migrations/20260508010000_personalization_improvements.sql`

- `profiles.onboarding_completed BOOLEAN DEFAULT false` 추가
- 기존 사용자 자동 마이그레이션: `primary_region_id IS NOT NULL AND age_band IS NOT NULL` → `onboarding_completed = true`
- `get_recommended_posters` 함수 재작성:
  - 지역 매칭: +40점
  - 카테고리 매칭: +30점
  - 대상 그룹/성별 매칭: +20점 (미매칭 시 -15점, 대상 없으면 0점)
  - 마감 임박 (7일 이내): +10점

### 온보딩 강제 적용
**파일:** `apps/web/middleware.ts`
- `onboarding_completed === false`이면 `/onboarding`으로 리디렉션
- role + onboarding_completed 단일 DB 쿼리로 처리

### 온보딩 5단계로 확장
**파일:** `apps/web/app/onboarding/page.tsx`
- 기존 4단계 → 5단계: 동의 → 지역 → 나이 → **성별** → 카테고리
- 완료 시 `gender` + `onboarding_completed: true` 저장

### 프로필 수정에 성별 추가
**파일:** `apps/web/app/mypage/edit/page.tsx`
- 성별 선택 UI 추가 (남성 / 여성 / 밝히고 싶지 않음)

## 4. 크롤러 구축 (scripts/crawler/)

### 배경
Claude.ai에서 생성한 `posterlink-crawler.tar.gz`를 검토하고, 실제 PosterLink DB 스키마에 맞게 수정하여 프로젝트에 통합.

### 구조
```
scripts/crawler/
├── src/
│   ├── index.js              # 메인 실행 파일
│   ├── sites.js              # 34개 사이트 설정
│   ├── crawler.js            # 공통 크롤링 로직
│   ├── adapters/
│   │   ├── index.js          # 어댑터 라우터
│   │   ├── generic-board.js  # 범용 게시판 어댑터
│   │   └── mapo-gu.js        # 마포구청 전용 어댑터
│   └── upload-to-supabase.js # Supabase 업로드 스크립트
├── package.json
├── .env.example
└── .gitignore
```

### 크롤링 대상 (34개 사이트)
서울시청, 마포구청(공지+채용), 16개 동주민센터, 마포구의회, 마포구시설관리공단, 마포문화재단, 마포인재육성장학재단, 마포문화원, 마포구고용복지지원센터, 마포구노동자종합지원센터, 구립망원청소년문화센터, 구립도화청소년문화의집, 구립마포청소년문화의집, 마포장애인종합복지관, 마포노인복지센터, 마포구민체육센터, 한국전기안전공사, 마포구어린이급식관리지원센터, 마포복지재단

### upload-to-supabase.js 핵심 수정 사항
원본 스크립트의 스키마 불일치 문제 완전 재작성:
- `@supabase/supabase-js` 클라이언트 사용 (기존: plain fetch)
- 올바른 필드명: `source_org_name`, `summary_short`, `poster_status: 'review'`, `source_key`
- `source_key` 기반 upsert로 중복 방지
- `poster_links` 테이블에 원본 URL 연동
- `poster_categories` 테이블에 카테고리 UUID 연동
- `dotenv` 지원

### 실행 결과 (2026-05-08 첫 실행)
- 총 수집: **151건**
- 서울시청: 6건, 마포구청: 29건, 마포복지재단: 114건(공지+채용), 마포문화재단: 2건
- 도메인 없음/404/SSL 오류 사이트: 약 10개 (추후 URL 수정 필요)
- 동주민센터 16개: 현재 게시글 없음 (URL은 정상)

### 사용법
```bash
cd scripts/crawler
cp .env.example .env   # Supabase 키 입력
npm install
npm run crawl          # 전체 크롤링
npm run upload data/results/all_YYYY-MM-DD.json  # Supabase 업로드
```

## 5. GitHub Actions 일일 배치

**파일:** `.github/workflows/crawler.yml`

- 매일 **01:00 UTC (한국 10:00 KST)** 자동 실행
- 수동 실행 버튼 (`workflow_dispatch`) 지원
- 크롤링 → Supabase 자동 업로드 → 결과 파일 7일간 artifact 보관
- `NODE_TLS_REJECT_UNAUTHORIZED=0` 설정으로 자체서명 인증서 허용

### 필요한 GitHub Secrets
| Name | 설명 |
|------|------|
| `SUPABASE_URL` | Supabase 프로젝트 URL |
| `SUPABASE_KEY` | service_role 키 (anon 키 아님) |
| `CRAWLER_USER_ID` | 크롤러 봇 계정 UUID (선택) |
